## 1. 产品概述

PDIS 是一个个人决策辅助 Agent，帮助用户分析职场/人际关系/重大个人决策问题。它通过聊天理解场景，沉淀“人际关系档案库 + 人物侧写库 + 决策历史”，在用户需要可行性判断与行动建议时给出辅助结论，并支持持续学习用户偏好与关系网络。

PDIS 的产品目标不是“更聪明的问答机器人”，而是让 AI 具备可感知、可规划、可执行、可复盘的能力：先明确用户目标与约束，再主动检索与调用工具，最后基于证据给出建议并沉淀为可复用记忆。

## 2. 用户体系

- 支持注册/登录，每个账户拥有独立的数据域：对话、人物档案、侧写版本、历史记录、附件证据。
- 支持多设备登录（同一账号），会话与资料在同一账户内一致可见。

### 3.1 AI 对话交互界面

- 设计形式：类似千问的聊天界面，包含
  - 智能输入框（支持文字输入；语音输入作为增强项）
  - 快捷功能按钮区
  - 历史对话记录
  - 多媒体附件支持（图片、txt、md 等文本）
- 交互特点
  - 自然语言对话，无需复杂操作
  - 上下文记忆：同一会话内持续记忆；跨会话按需召回人物/事件/历史结论
  - 目标澄清：当目标或约束不清晰时，以对话方式逐步澄清（不是一次性“补字段”）
  - 可见执行：在生成结论前会展示“计划摘要 + 工具执行状态 + 引用记忆”（不展示敏感推理细节）

### 3.2 场景切换按钮（输入框底部）

用户可在输入框底部切换使用场景（按钮切换不影响普通闲聊；Agent 在后端仍可按需调用工具）：
- 智能分析：深度理解用户面临的复杂决策场景
- 关系管理：建立个性化的人际关系档案库
- 行为预测：通过人物侧写预测他人行为模式
- 决策辅助：提供基于数据与逻辑的行动建议

### 3.3 核心功能按钮（快捷入口）

1) 人际关系档案库
- 管理用户的重要人际关系网络
- 记录联系人基本信息、关系类型、互动历史
- 支持标签分类（同事、朋友、家人等）

2) 人物侧写分析
- 基于 MBTI 16 型人格模型生成侧写（含置信度与依据）
- 支持侧写版本化（画像会随时间与新证据更新）

3) 个人决策辅助
- 对当前场景给出：可行性判断、关键依据、行动步骤、风险点、话术建议
- 支持多轮澄清与复盘（为什么这样建议、引用了哪些历史）

## 4. 功能需求（按页面/模块）

### 4.1 登录/注册页

- 注册：邮箱（或手机号）+ 密码 + 昵称
- 登录：账号 + 密码
- 安全：登录态过期、登出、异常登录提示（MVP 可简化）

### 4.2 对话页（默认入口）

- 输入：文本输入；附件上传（图片/txt/md）；发送后支持流式输出
- 澄清对话：当目标/约束不清晰时，Agent 以多轮对话确认关键约束（时间、风险偏好、可接受的代价、涉及人物的关系边界等）
- 记忆注入：在生成回答前，系统自动检索相似人物/事件/历史记录作为上下文注入给 LLM，并在 UI 上显示“引用了哪些记忆”
- 工具调用：当需要确认人物画像或历史事件时，Agent 会主动调用工具查询数据库，而不是要求用户把资料复制粘贴进来
- 输出结构：可行性（可行/有条件可行/不可行）+ 置信度 + 关键依据 + 行动步骤 + 风险 + 可选话术
- 沉淀：自动把“人物/事件/结论”写入用户的长期记忆（人物库/事件库/历史记录 + 向量索引）

### 4.3 人际关系档案库页

- 列表：按最近更新时间排序；支持搜索与标签过滤
- 新建/编辑：人物称谓、角色、关系、标签；支持从描述快速建档
- 互动追加：以“时间-事件-结果/影响”的方式追加，不覆盖旧信息
- 侧写：展示最新侧写 + 历史版本；支持手动触发“重新侧写”

### 4.4 历史记录页

- 列表：按时间倒序；支持关键词检索
- 详情：展示完整结论、引用人物/事件、模型版本与证据摘要
- 复用：一键“基于此记录再次分析”（带入上下文并允许补充新信息）

## 5. 核心流程（多轮闭环）

1. 用户选择场景（按钮）并输入问题/上传证据。
2. Agent 识别目标、约束与不确定性，必要时进入澄清对话。
3. Agent 主动调用工具：检索人物档案、召回相似历史、解析附件证据，形成可用上下文。
4. Agent 输出计划摘要与工具执行状态，然后生成决策建议（可行性/置信度/依据/行动/风险/话术）。
5. 系统沉淀本次对话、引用记忆、工具调用轨迹与最终结论到历史记录与长期记忆库。

## 6. Agent 能力（产品侧约束）

### 6.1 任务拆解与计划摘要

- 系统在输出结论前，必须先输出计划摘要（高层步骤，可折叠）。

### 6.2 工具调用（Tool Calling）

系统应具备主动调用内部工具的能力，至少包括：
- 查询人物档案与侧写版本
- 查询历史记录（按关键词/相似度/时间范围）
- 写入/更新人物档案、追加互动事件
- 附件解析（图片 OCR/总结、文本抽取、证据入库）

### 6.3 记忆机制（RAG）

- 人物侧写、互动事件、历史结论与关键对话片段需可向量化并支持语义检索
- 输出中必须标注引用来源（人物/事件/历史记录的引用列表），用于复盘与纠错
