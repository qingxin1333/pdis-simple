# 四、遇到的问题与处理（逐条）

1. **`externally-managed-environment`（pip 无法在系统 Python 安装包）**
   - 原因：未激活 virtualenv。
   - 解决：创建并 `source .venv/bin/activate` 使用 venv。
2. **`InvalidRequestError: Attribute name 'metadata' is reserved`**
   - 原因：模型中把列名取为 `metadata` 与 `Base.metadata` 冲突。
   - 解决：改为 `metadata_json = Column("metadata_json", JSONB, ...)`，或 Python 属性名使用 `metadata_`。
3. **`SAWarning: Did not recognize type 'vector'`**
   - 原因：缺少 `pgvector` Python 包。
   - 解决：`pip install pgvector` 并在 model 中 `from pgvector.sqlalchemy import Vector`。
4. **`CompileError: No literal value renderer ... REGCONFIG`（to_tsvector('simple', name)）**
   - 原因：SQLAlchemy 在编译索引表达式时无法把 'simple'（REGCONFIG）当作字面值渲染。
   - 解决：在 model 中用 `sa.text("to_tsvector('simple', name)")` 来声明索引表达式。
5. **`psycopg2.errors.InsufficientPrivilege: must be owner of table`**
   - 原因：自动生成的 migration 试图更新 table comment 等需要表 owner 的操作，而当前 DB 用户不是 owner。
   - 解决方案：你选择了把表 owner 批量改为 `pdis_master`（在 Navicat 执行了 DO $$ ... ALTER TABLE ...），并最终成功解决权限问题。
6. **migration 链断（缺失某个 revision 文件）**
   - 处理：你先备份旧文件并清空 `alembic/versions/`，然后重新生成 `initial migration` 并 `stamp head`。这是一个合理的 baseline 重建策略。
